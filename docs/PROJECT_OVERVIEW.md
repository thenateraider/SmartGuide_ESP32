# Project Overview

เอกสารนี้สรุปจากไฟล์ `หลักการทำงานโปรเจกต์SmartGuide.docx` เพื่อให้อ่านง่ายบน GitHub

## เป้าหมาย

Smart Guide เป็นระบบนำทางผู้ป่วยภายในโรงพยาบาล โดยใช้ ESP32-S3 ทั้งฝั่ง Mobile และ Station ร่วมกับ BLE RSSI, NFC, HTTP, หน้าจอ และระบบเสียง

## องค์ประกอบ

- Station1: ลงทะเบียนผู้ป่วย เปิดเว็บ `http://station1.local` และเป็น API กลาง
- Station2: จุดคัดกรอง ใช้ NFC เพื่อเปลี่ยน step จากไปคัดกรองเป็นกำลังคัดกรอง และจากกำลังคัดกรองเป็นไปพบแพทย์
- Station3: ห้องตรวจ ใช้ NFC เพื่อเปลี่ยน step เป็นอยู่กับแพทย์ และออกจากห้องตรวจไปชำระเงิน
- Station4: จุดชำระเงิน ใช้ปุ่มกดเพื่อยืนยันการชำระเงิน
- Station5: จุดรับยา ใช้ปุ่มกดเพื่อยืนยันรับยา เฉพาะเส้นทาง `m1`
- MobileWalk: อุปกรณ์ผู้ป่วย แสดงคำแนะนำ สแกน BLE เพื่อประมาณระยะ และเล่นเสียงตามสถานะ

## การสื่อสาร

- MobileWalk ดึงสถานะจาก Station1 ผ่าน `GET /status`
- Station2-5 อัปเดตสถานะผ่าน `GET /setstep?value=<step>`
- Station2-5 ส่ง BLE beacon ให้ MobileWalk ใช้วัดระยะจาก RSSI
- Station1 ให้บริการเว็บลงทะเบียนและแสดงสถานะผู้ป่วย

## Step หลัก

| Step | ความหมาย |
| --- | --- |
| 0 | ยังไม่มีผู้ป่วยลงทะเบียน |
| 1 | ลงทะเบียนแล้ว กำลังไปจุดคัดกรอง |
| 2 | กำลังคัดกรอง |
| 3 | คัดกรองเสร็จแล้ว กำลังไปห้องตรวจ |
| 4 | อยู่ที่ห้องตรวจ |
| 5 | พบแพทย์เสร็จแล้ว กำลังไปชำระเงิน |
| 6 | ชำระเงินแล้ว กำลังไปรับยา หรือจบสำหรับ `m2` |
| 7 | รับยาและคืนอุปกรณ์แล้ว |
| 8 | รอแตะ NFC หลังลงทะเบียน |
| 9 | ชำระเงินและคืนอุปกรณ์แล้วสำหรับกายภาพ |

## การใช้งานหน้างาน

1. เปิด WiFi ชื่อ `SmartGuide` รหัส `88888888`
2. เสียบไฟ Station แต่ละจุดหลัง WiFi พร้อมใช้งาน
3. เปิด MobileWalk ด้วยปุ่มเล็กใต้ช่อง SD Card
4. เข้า `http://station1.local` ด้วย Safari หรือใช้ IP ของ Station1 ถ้าอุปกรณ์ไม่ resolve mDNS
5. ลงทะเบียนชื่อผู้ป่วย เลือกแผนก และแตะ NFC เพื่อเริ่ม flow
